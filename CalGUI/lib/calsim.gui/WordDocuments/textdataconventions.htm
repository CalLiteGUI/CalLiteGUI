Text Data Conventions

Text data is defined as generic alpha-numeric lines of text where each line is preceded by a carriage return character and ends with a line feed character. It does not, at this time, include other types of characters such as those that would be used to create a graphical display. There are no definitive size limitations for a DSS text record but it is recommended that a record contain no more than about 1000 lines of text. There are no conventions set for the structure of the pathname; however, it is recommended that the pathname parts be labeled in a descending order of importance and that the pathname indicate that the record contains text data and not one of the other types of data.
