General FILE Conventions

 

Most “files” can be stored in an HEC-DSS database file.  These include .jpg files, .mp3 files, .pdf files, .xls files, and .doc files.  This allows the storage of various meta data along with regular data.  Photographs, fact sheets, sounds, etc. can be presented along with time series and paired data.

 

In HEC-DSsVue on Microsoft Windows computers, tabulating or plotting these types of meta data will launch the native application associated with the extension of that data set.  For example, “plotting” a pdf data set will launch Adobe Acrobat, a .xls data set will launch Excel, and so forth.  One exception is that an image file will launch a Java window with the image displayed.

 

Part A – Group

 

This part is for general grouping of data that would cause all records associated in a certain way to sort together, be copied together, delete together, or otherwise recognized as a group.

 

Part B – Location

 

This pathname part is the basic location identification of the data. If a photograph, it is the location of the picture.

 

Part C – File Name

 

This is the name of the file from disk, excluding any directory that it came from.

 

Part D – Meta Data Type

 

This part of the pathname can only be either FILE or IMAGE to reflect what the meta data is.

 

Part E – File Extension

 

This is the original extension of the filename.

 

Part F - General Descriptor

 

The F part identifies a unique descriptor of the data such as the situation, condition, or alternative plan name associated with the data.

 

Example Pathnames:

 

/SACRAMENTO/SHASTA/IMG_0009.JPG/IMAGE/JPG/SPILLWAY – AUG 2006/

 

/LOWER COLORADO/TRAVIS/TRAVIS.PDF/FILE/PDF/FACT SHEET/

 

/STONES/NO SECURITY/GIMME SHELTER.MP3/FILE/MP3/CARLS FAVORITES/
