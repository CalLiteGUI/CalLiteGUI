3.1. Run Settings

The Run Settings dashboard allows the user to load, save, run, and select scenario(s); view scenario settings; enter a scenario description;  batch runs; specify the run period, run type, and run basis.

Scenario settings are saved with a file extension .cls (CalLite Scenario). The file is portable and can be easily loaded by another user.  

The View Scenario Settings lets the user to view all the chosen options in a summarized table. This should be checked to ensure that all settings are correct before running the scenario(s).

The run period can be any period starting no earlier than Oct 1921 and ending no later than Sep 2003 (USGS water years 1922-2003).  In this version of CalLite, only deterministic run types are possible.  Probabilistic runs will be added in later versions of the model.  The run basis determines the type of CalSim II run that the CalLite run is based on.  Currently three run bases are available - Biological Opinion RPAs, Pre-Biological Opinion RPAs and D-1485.  The base CalSim II run primarily affects the AD (accretion/depletion) terms that are used in CalLite.  Sometimes there may be shortages associated with AD terms in the North of Delta area, and in this case CalLite will display a warning message at the end of the simulation.  See Appendix A of the CalLite Reference Manual for a further description of AD terms and shortages.

The Batch runs provides the capability to set up and batch run up to four CalLite studies, which could be more efficient than having to parameterize and run each individual study through a separate GUI.
